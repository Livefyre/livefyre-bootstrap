Version 2.x!

All the funs. None of the prefixes.

About This Version

  • This version of Livefyre Bootstrap was updated to support the creation of Asset Manager. Version 1.x still exists separately and will be maintained until the rest of Studio and our apps migrate to 2.x.
  • Version 2.x currently lives on the 2.x branch on Github. The 1.x version still lives on the master branch for now.
  • All of the content from the UI Components page has been included here for your convenience.
  • Fycons and their classes are pretty much identical to v1.x (minus the lf- prefix).

Usage

In your package.json or bower.json file, add or update the livefyre-bootstrap dependency: 

"livefyre-bootstrap": "git+ssh://git@github.com/Livefyre/livefyre-bootstrap.git#2.0.2"

Or, you can use the CDN urls:

https://cdn.livefyre.com/libs/livefyre-bootstrap/v2.0.2/livefyre-bootstrap.min.css
https://cdn.livefyre.com/libs/livefyre-bootstrap/v2.0.2/livefyre-bootstrap.min.js

Changes from v1.x

  • Removes the lf- prefix from all CSS classes, markup and JS. This makes it easier to update to the latest [Twitter] Bootstrap version, removes a lot of redundant CSS and allows us to take advantage of all [Twitter] Bootstrap features.
  • Updates the [Twitter] Bootstrap version to 3.3.4 (from 3.0.3).
  • [Twitter] Bootstrap 3.3.4's minimum required jQuery version is 1.9.1 (Bootstrap 3.0.3's minimum was 1.9.0).
  • Allows you to specify the location of your [Twitter] Bootstrap files.
  • New Slats styles.
  • New Alert Modal.
  • New Query Input (i.e. Search Form) styles.
  • Loader: Fixes bug where a one-pixel line sometimes appears between the second and third rows.
  • Many more listed alongside each section below.

Checklist for Migrating from v1.x

  1. Update your Livefyre Bootstrap files to 2.x. You may need to also update your version of [Twitter] Bootstrap to 3.3.4.
  2. Remove the lf- prefix from your CSS classes, markup and JS.
  3. Importing: In a variable named @bootstrap-less, specify the location of your [Twitter] Bootstrap files. Include this variable at the top of your main Less file or wherever you import Livefyre Bootstrap's variables and mixins files. (examples)
  4. Buttons: In your CSS, if you've added any custom vertical padding values to a button, you may need to remove or adjust them since buttons now use line-height instead of padding to determine height.
  5. Button Toolbars: Spacing between buttons and buttons groups has increased. You may need to adjust any subsequent layout breakage.
  6. Toggles: Update all toggle instances with the new markup. (example markup)
  7. Toggles: Toggles are now all-CSS so the toggles.js file has been removed. Be sure to remove any includes or references.
  8. Toggles: Update any JS which checks for state change on toggles. Event listeners should watch for changes to the input element. Don't use a click event listener because browsers may register two clicks: one on the label and one on the checkbox inside. (example markup)
  9. Dropdown Triggers: Caret icons have been changed from solid triangles to chevrons. Feel free to continue using the .caret class or switch to the .fycon-arrow-down class.
  10. Modals: Padding has increased all around. Check that this hasn't broken any modal layouts.
  11. Modals: The Small modal class (.modal-sm) has been increased from 300px to 450px wide. Use .modal-xs for 300px wide version.
  12. Modals: Use .text-left, .text-right and .text-center to align headlines, text and buttons.
  13. Modals: Replace instances of the .centered class (deprecated in Bootstrap) in modal bodies with .text-center.
  14. Modals: Remove the .animated class. Use the default .fade class in the .modal wrapper.
  15. Colors: All grey values have changed. Make sure your text is still readable, backgrounds aren't too dark/light, etc. (more)
  16. Popovers: The Light theme (i.e. the original theme) is now the default. No need to use the .light class anymore. The Dark theme is still available using the combination of .popover .dark.
  17. Tabs: Check tab navigation as their styles have been updated.

Importing Variables & Mixins

Changes from v1.x

Changes from v1.x

  • You must specify the location of your [Twitter] Bootstrap files using the @bootstrap-less variable.

Some of Livefyre Bootstrap's Less files rely on a variable called @bootstrap-less, which contains the location of your [Twitter] Bootstrap files. This allows you to situate your library files wherever you wish. When you're importing Livefyre Bootstrap's Less files individually, you may need to provide a value for this variable as there is no default fallback (couldn't figure this out, sry).

Example: In your main Less file, if your modules have their own variables and mixins files:

@bootstrap-less: "/node_modules/bootstrap/less";
@import "variables.less";
@import "mixins.less";

Example: Import LF-BS variables and mixins into your module's Less file:

@bootstrap-less: "/node_modules/bootstrap/less";
@import "/node_modules/livefyre-bootstrap/src/styles/variables.less";
@import "/node_modules/livefyre-bootstrap/src/styles/mixins.less";

Example: If you want to import all LF-BS files AND your [Twitter] Bootstrap files live inside /lib/bootstrap/less/, you can just import all of LF-BS's variables, mixins and partials in one import, since all.less contains a @bootstrap-less declaration:

@import "/lib/livefyre-bootstrap/all.less";

Slats

Use slats to create dashboard lists. Ex: Streams List, Users List

Size options

Use alongside the .slat class (but not with .slat-header).

  • Default - 60px high
  • .slat-sm - 48px high
  • .slat-lg - 72px high, larger title font

Other modifiers

  • .slat-vertical-center - Vertically centers content (default: top-aligned).
  • .slat-header - Header row.
  • .slat-variable - Height isn't fixed. For when you need to fit a varying amount of content.
  • .slat-hover - Adds hover styles. For when the entire slat is clickable.
  • .slat-disabled - Displays a disabled cursor on hover.
  • .no-bottom-border - Removes the bottom border. Ex: The bottom slat in a series.
  • .add-top-border - Adds a top border. Ex: The top slat in a series.

Titles

  • Use .slat-title to make content more prominent (ex: the name of the streams in the Streams list).
  • Since it applies display: inline-block, it can be used with a variety of HTML elements.
  • Or, you can add it to the .slat div directly.
  • Longer titles are truncated with ellipsis.

Toggles

Changes from v1.x

Changes from v1.x

  • Completely different markup.
  • All-CSS now. No more JS.
  • Uses checkbox for better accessibility and semantics.
  • Check for state change by using the checked attribute of input element (instead of .active class).
  • "On/Off" text lives in markup now so easier to swap in internationalization text.
  • Works inside of dropdown menus.
  • Visually unchanged from prevous version.

Labels

You can display text labels by adding the .toggle-label class, a data-off attribute and text inside of the span for the "on" state.

Sizing

There is an optional smaller toggle size which can be enabled by adding class .toggle-sm.

State

You can set a toggle to its "on" state by adding the checked attribute to the input element.

Event Listeners

Note that your event listeners should watch for changes to the input element. Don't use a click event listener because browsers may register two clicks: one on the label and one on the checkbox inside.

<label class="toggle">
  <input type="checkbox" role="checkbox" />
  <span></span>
</label>

<label class="toggle toggle-label">
  <input type="checkbox" role="checkbox" />
  <span data-off="Off">On</span>
</label>

<label class="toggle toggle-sm">
  <input type="checkbox" role="checkbox" checked />
  <span></span>
</label>

<label class="toggle toggle-sm toggle-label">
  <input type="checkbox" role="checkbox" checked />
  <span data-off="Off">On</span>
</label>

Tooltips

Changes from v1.x

Changes from v1.x

  • We extended the plugin so that after triggering a tooltip via click, a 2nd click anywhere - except on the tooltip itself - closes it. This overrides the default behavior where you can only close the tooltip by clicking the trigger element again.

Usage

For performance reasons, tooltip() is not loaded by default. You must initialize them yourself. You can do this via jQuery:

data-attribute: $('[data-toggle="tooltip"]').tooltip();

class/id selector: $('.tooltip').tooltip();

This is a tooltip. You can make them appear in a number of ways. Also to the left and down below.

Also to appear on click instead of on hover (a 2nd click anywhere - except on the tooltip itself - closes it).

<-- HTML -->
<span data-toggle="tooltip" data-title="Looks like you've found meh">tooltip</span>
<span data-toggle="tooltip" data-placement="bottom" data-title="Australia">below</span>
<span data-toggle="tooltip" data-trigger="click" data-title="*click*">appear on click</span>

<-- JS -->
$('[data-toggle="tooltip"]').tooltip();

$('#element').tooltip(
  {
    animation: true, // boolean
    html: false, // boolean; allows HTML to be inserted into the tooltip
    placement: 'top|bottom|left|right|auto', 
    trigger: 'click|hover|focus',
    title: 'string' // value if title attribute is not present on the element
    delay: int, // delay animation in ms
    container: 'string', // appends tooltip to a specific element
  }
);

Methods/Options

There are also a number of placement options to pass to tooltips. To access these, either configure them via the HTML5 Data API or in a javascript config:

data-attribute: <span data-toggle="tooltip" data-placement="top|bottom|left|right|auto">

Popovers

Usage

Popovers are syntatically similar to tooltips. In order to use popovers, you must include the tooltip.js lib as well as it is a dependency. Again, you are responsibile for initializing popovers yourself, and can access them via data-attribute or a selector.

<button class='btn btn-primary btn-lg'
  data-toggle="popover" 
  data-placement="top" 
  data-content="Celebrate Mark Doten Day when the weather is hotter than 90°F!">
  Click me!
</button>

Modals

Changes from v1.x

Changes from v1.x

  • Increased overall padding. Other minor style changes.
  • To add a new 450px modal size, we changed .modal-sm from 300px wide to 450px. You can use the new .modal-xs class to create 300px wide modals.
  • Make the bounce-in animation the default. So, no need to use the .animated class anymore.
  • Added ability to divide content inside the modal body using section.
  • Added support for [Twitter] Bootstrap's helper classes for text alignment (.text-left, .text-right, .text-center) so they can be used on the modal's title, body and footer containers. Note that the .centered class has been deprecated in [Twitter] Bootstrap.
  • Fixed close button so it no longer affects header text positioning.

Alert Modal

  • Alternative modal style, useful for alert messages and notifications that require the user's attention.
  • To use, add the .modal-alert class to .modal or .modal-dialog. You can't yet create an Alert Modal via a data attribute or JavaScript option.
  • Do not include a .modal-header section.
  • In .modal-body, use an h4 for the headline.
  • Everything is center-aligned by default but you can use [Twitter] Boostrap's helper classes .text-left, .text-right and .text-center to independently align headlines, text and buttons.

Usage

Include a template at the bottom of the page with class fade to hide it by default. Assign an id to this modal template and assign it as the data-target.

<button class="btn btn-lg btn-default" data-toggle="modal" data-target="#lfModal">
  Launch Modal
</button>

Size options

You can also pass additional modal sizes in lieu of the default 600px width on .modal-dialog:

  • .modal-xs: 300px
  • .modal-sm: 450px
  • .modal-md: 600px
  • .modal-lg: 900px

Alignment

Use .text-left, .text-right and .text-center on any element align headers, text and buttons.

Scrollable

Turn an element (ex: .list-group) into a scrollable container. Similar to .pre-scrollable in code.less.

Default max-height is 350px. Use additional classes to change size.
.scrollable-sm = 200px
.scrollable-lg = 500px

Loader

Usage

Livefyre's loader depends on loader.js (included in the minified js).

Options

loader accepts two parameters: elem and size (optional).

  • elem expects an object or jQuery object
  • size expects an integer or one of three pre-canned sizes as a string: 'mini', 'small', or 'large'.

<-- HTML -->
<div id="lfLoader"></div>

<-- JS -->
loader.decorate($('#lfLoader'), 120)
loader.decorate($('#lfLoader-large'), 'large')
loader.decorate($('#lfLoader-small'), 'small')
loader.decorate($('#lfLoader-mini'), 'mini')

Spinner

The generic spinner is for situations when we don't want Livefyre branding and colors (ex: an app that uses a client's branding).

Size options:

  • .spinner-xs = 16x16
  • .spinner-sm = 24x24
  • .spinner (default) = 36x36
  • .spinner-lg = 64x64
<!-- 16 x 16 -->
<div class="spinner spinner-xs"></div>

<!-- 24 x 24 -->
<div class="spinner spinner-sm"></div>

<!-- 36 x 36 (Default) -->
<div class="spinner"></div>

<!-- 64 x 64 -->
<div class="spinner spinner-lg"></div>
