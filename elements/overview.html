Livefyre Bootstrap v2.0.2

Background

  • This version of Livefyre Bootstrap was updated to support the creation of Asset Manager.
  • Version 1.x still exists separately and will be maintained until the rest of Studio and our apps migrate to 2.x.

Usage

In your package.json or bower.json file, add or update the livefyre-bootstrap dependency:

"livefyre-bootstrap": "git+ssh://git@github.com/Livefyre/livefyre-bootstrap.git#v2.0.2"

Or, you can use the CDN urls:

https://cdn.livefyre.com/libs/livefyre-bootstrap/v2.0.2/livefyre-bootstrap.min.css
https://cdn.livefyre.com/libs/livefyre-bootstrap/v2.0.2/livefyre-bootstrap.min.js

Contributing

  • Version 2.x currently lives on the 2.x branch on Github.
  • Version 1.x still lives on the master branch for now.
  • Do NOT push directly to either of these branches. Instead, please create a feature-branch and submit a pull request. Jenkins watches these branches to execute builds, tests and to generate CDN files. If merges are not done properly, there will be breakage.

What's New in v2.x

  • Removes the lf- prefix from all CSS classes, markup and JS. This makes it easier to update to the latest [Twitter] Bootstrap version, removes a lot of redundant CSS and allows us to take advantage of all [Twitter] Bootstrap features.
  • Updates the [Twitter] Bootstrap version to 3.3.4 (from 3.0.3).
  • [Twitter] Bootstrap 3.3.4's minimum required jQuery version is 1.9.1 (Bootstrap 3.0.3's minimum was 1.9.0).
  • Allows you to specify the location of your [Twitter] Bootstrap files.
  • New Slats styles.
  • New Alert Modal.
  • New Query Input (i.e. Search Form) styles.
  • Many more listed alongside each section.

Checklist for Migrating from v1.x

  1. Update your Livefyre Bootstrap files to 2.x. You may need to also update your version of [Twitter] Bootstrap to 3.3.4.
  2. Remove the lf- prefix from your CSS classes, markup and JS.
  3. Importing: In a variable named @bootstrap-less, specify the location of your [Twitter] Bootstrap files. Include this variable at the top of your main Less file or wherever you import Livefyre Bootstrap's variables and mixins files. (examples)
  4. Buttons: In your CSS, if you've added any custom vertical padding values to a button, you may need to remove or adjust them since buttons now use line-height instead of padding to determine height.
  5. Button Toolbars: Spacing between buttons and buttons groups has increased. You may need to adjust any subsequent layout breakage.
  6. Toggles: Update all toggle instances with the new markup. (example markup)
  7. Toggles: Toggles are now all-CSS so the toggles.js file has been removed. Be sure to remove any includes or references.
  8. Toggles: Update any JS which checks for state change on toggles. Event listeners should watch for changes to the input element. Don't use a click event listener because browsers may register two clicks: one on the label and one on the checkbox inside. (example markup)
  9. Dropdown Triggers: Caret icons have been changed from solid triangles to chevrons. Feel free to continue using the .caret class or switch to the .fycon-arrow-down class.
  10. Modals: Padding has increased all around. Check that this hasn't broken any modal layouts.
  11. Modals: The Small modal class (.modal-sm) has been increased from 300px to 450px wide. Use .modal-xs for 300px wide version.
  12. Modals: Use .text-left, .text-right and .text-center to align headlines, text and buttons.
  13. Modals: Replace instances of the .centered class (deprecated in Bootstrap) in modal bodies with .text-center.
  14. Modals: Remove the .animated class. Use the default .fade class in the .modal wrapper.
  15. Colors: All grey values have changed. Make sure your text is still readable, backgrounds aren't too dark/light, etc. (more)
  16. Popovers: The Light theme (i.e. the original theme) is now the default. No need to use the .light class anymore. The Dark theme is still available using the combination of .popover .dark.
  17. Tabs: Check tab navigation as their styles have been updated.
  18. Fycons: Identical to v.1. All fycons appear in both versions.
