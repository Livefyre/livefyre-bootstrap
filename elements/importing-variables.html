Importing Variables & Mixins

{% include btn-reference.html target="importing-variables" %}

What's New in v2.x

  • You must specify the location of your [Twitter] Bootstrap files using the @bootstrap-less variable.

Some of Livefyre Bootstrap's Less files rely on a variable called @bootstrap-less, which contains the location of your [Twitter] Bootstrap files. This allows you to situate your library files wherever you wish. When you're importing Livefyre Bootstrap's Less files individually, you may need to provide a value for this variable as there is no default fallback (couldn't figure this out, sry).

Example: In your main Less file, if your modules have their own variables and mixins files:

{% highlight css %}@bootstrap-less: "/node_modules/bootstrap/less"; @import "variables.less"; @import "mixins.less";{% endhighlight %}

Example: Import LF-BS variables and mixins into your module's Less file:

{% highlight css %}@bootstrap-less: "/node_modules/bootstrap/less"; @import "/node_modules/livefyre-bootstrap/src/styles/variables.less"; @import "/node_modules/livefyre-bootstrap/src/styles/mixins.less";{% endhighlight %}

Example: If you want to import all LF-BS files AND your [Twitter] Bootstrap files live inside /lib/bootstrap/less/, you can just import all of LF-BS's variables, mixins and partials in one import, since all.less contains a @bootstrap-less declaration:

{% highlight css %}@import "/lib/livefyre-bootstrap/all.less";{% endhighlight %}
